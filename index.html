Introduction

React

A JavaScript library for building user interfaces

React is a JavaScript library with the purpose to simplify the development of visual interfaces.

According to Flavio (see below), its primary goal is to make reasoning about an interface and its state at any point in time easy by dividng the UI into a collection of components.

It was developed at Facebook and powers both Facebook and Instagram as well as numerous other applications. It was publicly released in 2013.

The popularity of React

Ever since its release into the world, React has enjoyed widespread popularity for three main reasons:

  • Simplicity of use, also for existing projects
  • Ideal release date
  • Facebook loves it
Let's dive into each of them in turn.

Less complex than the other alternatives

In contrast to the libraries that existed before (Ember.js and Angular 1.x), React was purposefully designed so it would be easy to integrate it into existing projects. Facebook had to do it this way in order to introduce it to their existing codebase. Additionally, React's creators decided to only implement the View Layer. Ember.js and Angular 1.x, by contrast, the predominant frameworks at the time, not only used the full MVC stack but also imposed a lot conventions on any projects that wanted to use them. Using them in an existing project wasn't convenient at all.
For information on the Model-View-Controller (MVC) architecture see Martin Fowler's website.

Perfect timing

Additionally, when React was published, Google had just announced Angular 2.x and the new Angular framework was so wildly different from Angular 1.x that moving from 1 to 2 was "like moving to a different framework". React also provided a certain execution speed that developers were eager to try.

Backed by Facebook

Being backed by Facebook obviously is going to benefit a project if it turns out to be successful. Facebook currently has a strong interest in React, sees the value of it being Open Source, and this is a huge plus for all the developers using it in their own projects.
